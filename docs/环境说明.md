# 环境说明

本仓库不上传完整 Conda/venv 环境目录。原因是环境目录体积大、包含大量二进制依赖和缓存，不适合 GitHub，也不适合网页 ChatGPT 阅读。

## 已知环境口径

- VBench 版本：官方 VBench v1，`Vchitect/VBench` 的 `v0.1.5`
- Python：从环境目录判断为 Python 3.11
- 评测环境原始目录：`/data/zengxin/zwt/conda_envs/vbench_v1`
- ConsisID 生成环境原始目录：`/data/zengxin/zwt/conda_envs/face_consisid`
- 当前整理后的服务器项目根目录：`/data/zengxin/zwt/face_video`

## 为什么不上传环境压缩包

- GitHub 不适合保存完整 Python 环境、site-packages、模型缓存和编译产物。
- 上一次 push 失败的重要原因之一是仓库包体过大，而不是报告文件本身过大。
- 环境问题应通过版本、依赖清单、安装日志和失败日志复现，而不是上传整个环境目录。
